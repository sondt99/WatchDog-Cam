################################################################################
# PHẦN A: ADD-ON "SIMPLE AI VISION" (thư mục cũ: simple_ai_vision/)
################################################################################

TỔNG QUAN KIẾN TRÚC

Add-on Home Assistant nhẹ (phiên bản cuối 1.4.16, slug simple_ai_vision), viết
bằng Python, gồm 2 file mã nguồn: app.py (backend FastAPI, toàn bộ logic) và
ui.py (chỉ chứa 1 hằng chuỗi INDEX_HTML là trang web single-page HTML/CSS/JS
thuần). Luồng chính: Frigate phát hiện người qua MQTT -> Home Assistant
Automation gọi POST /analyze -> add-on lấy snapshot JPEG từ go2rtc (fallback
Frigate) -> gửi ảnh base64 tới API vision kiểu OpenAI-compatible -> match
keyword/regex trên câu trả lời -> nếu match thì gửi ảnh + phân tích qua
Telegram sendPhoto -> ghi sự kiện vào file JSONL. Add-on KHÔNG tự polling
camera, không decode RTSP, không ffmpeg, không database, không object detection
cục bộ.

A1. TÍCH HỢP AI / VISION MODEL

A1.1. Endpoint và xác thực
- URL gọi vision: "{ai_base_url}/chat/completions" (ai_base_url được rstrip
  dấu "/" trước khi nối). Mặc định ai_base_url = "https://api.openai.com/v1",
  tức URL đầy đủ mặc định là https://api.openai.com/v1/chat/completions.
- Model mặc định: "gpt-4o-mini". Hỗ trợ mọi gateway OpenAI-compatible:
  OpenAI, OpenRouter, 9Router, Gemini qua gateway (placeholder UI gợi ý
  "http://9router.local:20128/v1" và model "cc/claude-opus-4-7").
- Headers gửi đi (HTTP POST, thư viện requests, json=payload):
    Authorization: Bearer {ai_api_key}
    Content-Type: application/json
    Accept: application/json
- Timeout: ai_timeout giây (mặc định 30, số nguyên >= 1). KHÔNG có retry —
  mỗi lần gọi chỉ 1 request, lỗi thì raise_for_status() và trả lỗi về caller.
  KHÔNG đặt max_tokens. Không streaming ("stream": false).

A1.2. Payload request vision (hàm call_ai)
{
  "model": "<ai_model>",
  "messages": [
    {
      "role": "user",
      "content": [
        {"type": "text", "text": "<prompt>"},
        {"type": "image_url", "image_url": {"url": "<data_url>"}}
      ]
    }
  ],
  "temperature": 0.2,
  "stream": false
}

A1.3. Payload request test AI (hàm call_ai_text, endpoint /api/test-ai)
Giống trên nhưng content là chuỗi thuần:
{"role": "user", "content": "Reply with OK only."}, "temperature": 0.

A1.4. Mã hóa ảnh
Ảnh snapshot JPEG đọc từ file tạm, base64-encode và nhét vào data URL dạng:
"data:image/jpeg;base64,<base64>". Không resize, không nén lại.

A1.5. Prompt mặc định (DEFAULT_PROMPT, nguyên văn)
"Bạn đang phân tích ảnh camera an ninh.
Chỉ mô tả các sự kiện quan trọng liên quan đến an ninh.
Nếu không có gì quan trọng hãy trả lời NORMAL."

Cơ chế "Prompt Profiles": danh sách các object {"title": "...", "prompt":
"..."} lưu trong config; mỗi camera có trường prompt_profile trỏ tới title;
nếu camera có gán thì dùng prompt của profile, nếu không dùng prompt mặc định.
Validation: title và prompt bắt buộc đi cặp, title không trùng.

Prompt gợi ý trong README (nguyên văn):
"Bạn là hệ thống phân tích ảnh camera trong nhà.

Nếu thấy người trong ảnh, chỉ trả lời:
ALERT_PERSON: mô tả ngắn trong tối đa 20 từ.

Nếu không thấy người, chỉ trả lời:
NORMAL

Không giải thích.
Không hướng dẫn.
Không viết code.
Không nhắc đến Telegram."

A1.6. Parse response AI
- Hàm response_ai_content kiểm tra Content-Type: nếu chứa "text/event-stream"
  HOẶC body bắt đầu bằng "data:" thì parse theo SSE, ngược lại parse JSON.
- Parse JSON thường: lấy data["choices"][0]["message"]["content"]; nếu content
  là list (multi-part) thì ghép các part.get("text") bằng "\n"; strip kết quả.
  Lỗi cấu trúc -> ValueError "invalid AI API response".
- Parse SSE: duyệt từng dòng bắt đầu "data:", bỏ "[DONE]", json.loads payload,
  với mỗi choice lấy choice["delta"]["content"] (ưu tiên) hoặc
  choice["message"]["content"], nối tất cả; nếu rỗng raise ValueError.
- Mapping lỗi provider: 404 -> "AI API endpoint not found. Check ai_base_url
  and provider path."; 401 -> "AI API unauthorized. Check ai_api_key.";
  403 -> forbidden; 429 -> rate limited/quota; kèm provider_status và 1000 ký
  tự đầu body provider trong JSON trả về.

A1.7. Keyword matching (hậu xử lý AI)
Keyword mặc định: ["person", "human", "stranger", "fire", "smoke", "người",
"cháy"]. Mỗi keyword thử như regex re.search(pattern, analysis, IGNORECASE);
nếu regex lỗi thì fallback so sánh substring lowercase. Trả về keyword đầu
tiên match. Match -> gửi Telegram; không match -> chỉ ghi event "no_match".

A2. XỬ LÝ CAMERA / SNAPSHOT

A2.1. Nguồn ảnh chính: go2rtc
GET {go2rtc_url}/api/frame.jpeg?src={camera}, timeout snapshot_timeout (mặc
định 10s). Tên camera phải khớp regex ^[A-Za-z0-9_.:-]+$. Kiểm tra response
là ảnh: content-type chứa "image" HOẶC bytes bắt đầu \xff\xd8. Ảnh ghi ra
file tạm /tmp prefix "simple_ai_vision_{camera}_" đuôi .jpg, xóa sau khi dùng.

A2.2. Fallback: Frigate API
GET {frigate_url}/api/{camera}/latest.jpg (cùng timeout, cùng kiểm tra ảnh).

A2.3. Tự động dò URL (auto-discovery)
- resolve_go2rtc_url: dùng go2rtc_url cấu hình nếu có; nếu không, thử các
  candidate go2rtc của Frigate bằng GET {candidate}/api/streams (timeout kẹp
  1..5s) tới khi thành công.
- Candidates go2rtc (port 1984): (a) từ frigate_url tách scheme+host bằng
  regex ^(https?://[^/:]+)(?::\d+)? rồi ghép ":1984"; (b) host lấy từ
  Supervisor API dạng http://{host}:1984; (c) hardcode:
  http://ccab4aaf-frigate:1984, http://ccab4aaf_frigate:1984,
  http://frigate:1984. (ccab4aaf là prefix slug Frigate addon chính thức.)
- Candidates Frigate API (port 5000): tương tự với :5000.

A2.4. Liệt kê stream (GET /api/go2rtc/streams) — thử theo thứ tự:
  1) GET {go2rtc_url}/api/streams (source="go2rtc")
  2) GET {candidate:1984}/api/streams (source="Frigate go2rtc")
  3) GET {candidate:5000}/api/go2rtc/streams (source="Frigate go2rtc proxy")
  4) GET {candidate:5000}/api/config (source="Frigate config") — parse
     data["go2rtc"]["streams"] và data["cameras"].

A2.5. Live view UI: chỉ là ảnh snapshot tự refresh (img đổi src 1.5s trong
viewer / 5s ở grid; mode khác 3s/8s), hoặc iframe nhúng
{go2rtc_base}/stream.html?src={camera}&mode=mse, hoặc ảnh
{frigate_base}/api/{camera}/latest.jpg?_={timestamp}.

A3. WEB UI, ENDPOINTS, TÍCH HỢP HA ADD-ON

A3.1. Framework: FastAPI 0.111.0 + uvicorn 0.30.1 + requests 2.32.3.
app = FastAPI(title="Simple AI Vision", docs_url=None, redoc_url=None).

A3.2. Endpoints:
- GET /health -> {"success": true}
- GET / -> HTMLResponse INDEX_HTML
- GET /api/config; POST /api/config (merge + validate + ghi atomic .tmp rồi
  os.replace; ai_api_key/telegram_bot_token rỗng thì GIỮ giá trị cũ)
- GET /api/go2rtc/streams?go2rtc_url=&frigate_url=
- GET /api/events — 100 dòng cuối /data/simple_ai_vision_events.jsonl, mới
  nhất trước, bỏ status "disabled"
- GET /api/camera/frame?camera=X — proxy JPEG, Cache-Control: no-store
- POST /api/test-ai — call_ai_text("Reply with OK only.")
- POST /api/test-telegram — sendMessage "Simple AI Vision Telegram test OK."
- POST /analyze — endpoint chính. Body {"camera": "<go2rtc_src>"}. Luồng:
  validate options (bắt buộc ai_api_key, ai_base_url, ai_model,
  telegram_bot_token, telegram_chat_id) -> nếu camera không có trong danh
  sách: {"skipped":true,"reason":"camera not configured"}; camera tắt:
  "camera disabled" -> snapshot (go2rtc, fallback Frigate) -> base64 data URL
  -> chọn prompt theo profile -> call_ai -> match keyword -> match: Telegram
  sendPhoto + event "sent" (lỗi -> "telegram_error"); không match: "no_match"
  -> trả {"success","matched","matched_keyword","analysis"}. Nhánh lỗi ghi
  event: config_error, timeout, upstream_error, network_error, internal_error.

A3.3. Event log: /data/simple_ai_vision_events.jsonl, mỗi dòng 1 JSON:
{"time": ISO8601 UTC, "status", "camera", "keyword", "analysis", "error"}.

A3.4. config.yaml addon: ingress: true, ingress_port 8000, ports 8000/tcp,
homeassistant_api: true, arch amd64+aarch64, options/schema RỖNG có chủ đích
(config lưu ở /data/simple_ai_vision_config.json qua Web UI).

A3.5. run.sh: exec uvicorn app:app --host 0.0.0.0 --port 8000.
Dockerfile: python:3.11-alpine, pip install requirements, copy 3 file.

A3.6. Ingress: JS client dùng hàm apiPath(path) ghép đường dẫn tương đối
trên window.location.pathname (thêm "/" cuối nếu thiếu, bỏ "/" đầu của path)
— chạy được cả dưới /api/hassio_ingress/<token>/ lẫn cổng 8000 trực tiếp.
Fetch có AbortController timeout: config 15-20s, analyze 90s, test-ai 60s,
test-telegram 30s, streams/events 15s.

A3.7. Các khóa config (/data/simple_ai_vision_config.json):
go2rtc_url="", frigate_url="", go2rtc_host_url="", frigate_host_url="",
ai_api_key="", ai_base_url="https://api.openai.com/v1",

B1. TÍCH HỢP AI VISION (ai.py)

B1.1. Endpoint và xác thực
- API chuẩn OpenAI-compatible chat completions. URL dựng từ config: lấy
  config["ai_base_url"], bỏ "/" cuối; nếu chưa kết thúc bằng
  "/chat/completions" thì nối thêm. Base mặc định
  "https://9router.minhhungtsbd.me/v1" ->
  "https://9router.minhhungtsbd.me/v1/chat/completions".
- Headers: "Authorization: Bearer <ai_api_key>", "Content-Type:
  application/json". requests.Session dùng chung, User-Agent
  "fall-detection-web/2.0". Timeout POST = 120 giây.

B1.2. Payload request (JSON)
{
  "model": "<tên model>",
  "messages": [
    {
      "role": "user",
      "content": [
        {"type": "text", "text": "<prompt>"},
        {"type": "image_url", "image_url": {"url":
          "data:image/jpeg;base64,<base64 của file ảnh>"}}
      ]
    }
  ],
  "max_tokens": 1000,
  "stream": false
}
- Ảnh: đọc bytes file JPEG, base64-encode, ghép data URL. Không resize trước
  khi gửi (ảnh đã được nén lúc lưu event, xem B4.2).

B1.3. Model + retry
- Model chính: config["vision_model"], mặc định "gh/oswe-vscode-prime".
- Fallback: config["fallback_vision_model"] (mặc định rỗng). Nếu model chính
  ném RuntimeError (lỗi HTTP) và fallback khác rỗng + khác model chính thì
  gọi lại đúng 1 lần với fallback, cùng prompt/ảnh.
- Đo latency bằng time.monotonic(), log "[AI] latency=%.2fs result=...".
- Lỗi HTTP: nếu body bắt đầu "<" hoặc content-type chứa "html" thì message =
  "HTML Response (status reason)"; raise RuntimeError
  f"{status_code} Error: {80-100 ký tự đầu body}".

B1.4. Prompt mặc định (DEFAULT_VERIFY_PROMPT, nguyên văn):
"""Bạn là hệ thống xác minh té ngã từ ảnh camera trong nhà.

Nhiệm vụ:
- Xác định người có bị té ngã, nằm bất thường dưới đất, gặp nguy hiểm, cần
trợ giúp, hoặc cố đứng dậy thất bại không.
- Nếu có nguy hiểm, dòng 1 chỉ trả lời: EMERGENCY
- Nếu bình thường, dòng 1 chỉ trả lời: SAFE
- Dòng 2 mô tả rất ngắn tình huống trong ảnh, tối đa 20 ký tự.

Chỉ trả lời đúng 2 dòng, không giải thích thêm:
SAFE hoặc EMERGENCY
Mô tả dưới 20 ký tự
"""
- Prompt theo camera: mỗi camera có "prompt_id" trỏ tới phần tử trong
  config["prompts"] (list {id, title, content}, id = uuid4 nếu thiếu).
  Prompt rỗng -> fallback "Please analyze this image.".

B1.5. Parse response (nhiều định dạng)
- Nếu content-type chứa "text/event-stream" hoặc body bắt đầu "data:" ->
  parse SSE; ngược lại parse JSON chuẩn; nếu JSON không hợp lệ -> parse
  "concatenated JSON" (nhiều object nối liền, json.JSONDecoder().raw_decode
  lặp).
- JSON chuẩn: data["choices"][0]["message"]["content"]; content là list thì
  gom các dict lấy key "text" nối "\n".
- SSE: từng dòng "data: ...", bỏ "[DONE]", gom text từ
  choices[i].message.content hoặc .delta.content hoặc .text.

B1.6. Xử lý "thinking" và verdict
- strip_thinking_content: xóa block <think>...</think> (IGNORECASE, DOTALL);
  xóa block <think> mở không đóng (đến hết chuỗi); xóa các dòng bắt đầu
  "THOUGHT FOR" hoặc "THINKING" (uppercase).
- parse_ai_verdict -> (result, description, raw):
  * Dòng đầu (sau strip thinking) bắt đầu "EMERGENCY" -> EMERGENCY;
    "SAFE" -> SAFE.
  * Không rõ: chuỗi uppercase chứa "EMERGENCY" -> EMERGENCY, chứa "SAFE" ->
    SAFE, mặc định fail-safe "SAFE".
  * Description: dòng 2 trở đi, bóc tiền tố "DESC:", "DESCRIPTION:",
    "MÔ TẢ:", "MO TA:", "-", "2."; bỏ dòng chỉ là SAFE/EMERGENCY; cắt tối đa
    100 ký tự (short_text).
  * raw = content gốc chưa strip thinking.

B1.7. Telegram
- send_telegram(photo_path, message, config): POST multipart
  https://api.telegram.org/bot<token>/sendPhoto, data {chat_id,
  caption=message}, files {photo}, timeout 60s.
- Nội dung cảnh báo: "Emergency detected by AI vision.\nCamera: <tên>" + nếu
  có mô tả "\nScene: <mô tả>".

B2. LUỒNG PHÁT HIỆN TÉ NGÃ / XÁC MINH (monitor.py)

B2.1. Kiến trúc
- Thread chính _monitor_loop + mỗi camera 1 thread capture_latest_frames ghi
  frame mới nhất vào holder {frame, time, seq} có lock riêng.
- Nguồn frame: (a) go2rtc: GET <go2rtc_url>/api/frame.jpeg?src=<src> — src
  RTSP thêm fragment "media=video" + "backchannel=0"; kèm param chống cache
  "_ts"=epoch ms; headers Accept image/jpeg, Cache-Control no-cache, UA giả
  Chrome/126; retry 3-4 lần sleep 0.25*attempt; decode cv2.imdecode.
  (b) RTSP trực tiếp: cv2.VideoCapture, CAP_PROP_BUFFERSIZE=1.
- Nhịp go2rtc: max(go2rtc_frame_interval (1.0s), loop_sleep). Lỗi đọc:
  exponential backoff min(2^n, 30s), event "stream_reconnect" tối đa 1/30s.

B2.2. Vòng phát hiện YOLO
- Model: ultralytics YOLO(config["yolo_model"]) — mặc định "yolov8n.pt",
  imgsz=416, conf=0.5, classes=[0] (chỉ người). Xử lý mỗi frame thứ N,
  N=frame_skip (2). Cuối vòng sleep loop_sleep (0.3s).
- Khi phát hiện người:
  a) Ghi hình: nếu teldrive bật + camera.teldrive_record_enabled + qua
     cooldown (record_cooldown, mặc định 300s) -> thread record_and_upload_clip.
  b) Xác minh AI: nếu now - last_verify > verify_interval (20s) -> ghi frame
     ra data/camera_<index>.jpg + data/latest.jpg -> verify_scene. Thành công
     -> event "verified" (confidence, ai_result, ai_raw=description,
     ai_response=raw). Thất bại -> "ai_error", coi ai_result="SAFE".
  c) ai_result == "EMERGENCY": nếu now - last_alert > alert_cooldown (300s)
     -> Telegram sendPhoto + event "telegram_sent" (lỗi "telegram_error");
     còn cooldown -> event "cooldown".
- Chỉ event "verified" + EMERGENCY mới upload ảnh lên Teldrive.

B2.3. Circuit breaker dịch vụ ngoài
- Đếm consecutive_ai_failures và consecutive_upload_failures riêng. >=3 lỗi
  liên tiếp: suspend đến now + backoff; backoff: <=3 -> 60s; 4 -> 300s;
  5 -> 900s; >=6 -> 3600s. Khi suspend: bỏ qua AI (coi SAFE) / bỏ qua upload.
- Khi suspend gửi Telegram: "⚠️ Cảnh báo: Dịch vụ AI Cloud liên tiếp lỗi
  (N lần). Hệ thống tạm ngưng gọi AI đến HH:MM:SS để tránh quá tải.\nLỗi:
  <exc>" (tương tự cho Teldrive). Hồi phục: "✅ Thông báo: Dịch vụ AI Cloud
  đã khôi phục hoạt động bình thường trên camera <tên>."

B2.4. Ghi clip sự cố
- Ưu tiên go2rtc: GET <base>/api/stream.mp4?src=<src video-only>&duration=
  <record_seconds+6>&filename=<tên>&video=h264 (cộng 6s bù keyframe; ép H.264
  để browser phát được), stream về file, timeout duration+30.
- Fallback: cv2.VideoWriter fps=8.0, codec thử avc1/H264/mp4v (.mp4) rồi
  MJPG (.avi). Tên: data/event_clips/<YYYYmmddTHHMMSS>_<safe_camera>.mp4.
- Upload xong tạo event "teldrive_video_uploaded" lưu teldrive_video_id/
  name/path + thumbnail (frame đầu clip, JPEG quality 78, <stem>_thumb.jpg).
- Bảo trì (thread riêng, mỗi 600s): retry upload clip chưa upload (chỉ clip
  cũ hơn 300s tránh race), xóa thumbnail mồ côi >600s, xóa clip local đã
  upload nếu camera.local_save_videos=False, đồng bộ thumbnail thiếu.
  Metadata clip parse từ tên file: 15 ký tự đầu UTC "%Y%m%dT%H%M%S".

B3. TÍCH HỢP TELDRIVE (teldrive.py)

B3.1. Cấu hình và auth
- Bật khi: teldrive_enabled == True và teldrive_token khác rỗng.
- Base URL: config["teldrive_base_url"] (mặc định
  "https://teldrive.minhhungtsbd.me"); tự nối "/api" nếu chưa có.
- Headers MỌI request (điểm quan trọng — fix lỗi 401): token gửi 2 nơi:
    Authorization: Bearer <token>
    Cookie: user-session=<token>; access_token=<token>

B3.2. Các endpoint
- Kiểm tra token: GET /api/auth/session, timeout 15. 401 -> "Token is
  expired or invalid"; 403 -> "Token is not allowed"; OK -> đọc name,
  userName, expires.
- Tạo thư mục: POST /api/files/mkdir body {"path": "<folder>"}, chấp nhận
  200/201/204/409 (409 = đã tồn tại).
- Upload (single-part):
  1) upload_id = uuid4().hex. POST /api/uploads/{upload_id}, query params:
     partName=<tên file>, fileName=<tên file>, partNo=1, encrypted="false",
     hashing="false", channelId=<int> nếu có. Headers: auth + Content-Type
     <mime> + Content-Length <size>. Body = raw bytes (stream từ file
     handle). Timeout 180s. Response JSON có partId, salt?, encrypted?.
  2) GET /api/uploads/{upload_id} lấy danh sách parts (rỗng thì dùng part từ
     bước 1). Sắp theo partNo, mỗi part -> {"id": int(partId), "salt": ...}.
  3) Commit: POST /api/files JSON: {"name", "type":"file", "path":
     "/<folder>/", "mimeType", "uploadId", "parts": [...], "size",
     "encrypted": bool, "channelId"?: int}. Response chứa id, name, path.
- MIME: .mp4 video/mp4, .avi video/x-msvideo, .jpg/.jpeg image/jpeg, .png
  image/png; fallback mimetypes.guess_type rồi application/octet-stream.
- Cấu trúc thư mục: root = teldrive_root_path (mặc định "/Fall Detection");
  ảnh: <root>/<tên_camera_sạch>/images; video:
  <root>/<tên_camera_sạch>/videos/<YYYY-MM-DD> (ngày parse từ 15 ký tự đầu
  tên file). Tên camera sạch: chỉ giữ chữ số/chữ/-/_/space, space -> "_".
- URL file: <base>/api/files/{file_id}/{tên file url-encoded quote safe=''}.
- Download: GET URL trên, header Range nếu có, stream=True, timeout 60; thử
  KHÔNG auth trước, 401/403 thì retry kèm auth headers.
- Prewarm: tải trước "bytes=0-524288" (512KB đầu) đánh thức cache stream
  Teldrive/Telegram, tránh timeout khi phát video.

B3.3. Proxy và mapping thumbnail
- Route GET /api/teldrive/file/{file_id}/{file_name} (cần đăng nhập): ảnh
  cache đĩa data/teldrive_cache/<file_id>, Cache-Control private
  max-age=86400 + ETag/Last-Modified/304; video stream passthrough (giữ
  Accept-Ranges, Content-Length, Content-Range, ETag; Content-Disposition
  inline). Teldrive trả 401/403 -> redirect 302 đến URL Teldrive gốc.
- Mapping thumbnail event "verified" thiếu ảnh: tìm trong bảng events record
  cùng camera, status='teldrive_video_uploaded', có teldrive_image_id, time
  trong ±120 giây quanh event; thấy thì image_url =
  "/api/teldrive/file/<id>/<name-encoded>". Ưu tiên: teldrive_image_id/name
  -> proxy teldrive; không thì ảnh local "/api/event-image/<file>"; cuối
  cùng dynamic-match.
- /api/recordings: video_url = URL Teldrive trực tiếp, video_proxy_url =
  đường proxy nội bộ.

B4. REDIS, DB SCHEMA, AUTH

B4.1. Redis (fail-safe, tắt/lỗi thì bỏ qua cache)
- redis.ConnectionPool(host, port, db, password hoặc None,
  socket_timeout=2.0, socket_connect_timeout=2.0, decode_responses=True);
  pool cache theo tuple config; mỗi get_client đều ping(), lỗi -> None.
- API: get_cache/set_cache(key, value, expire)/delete_cache/
  clear_cache_pattern (SCAN count=100, không dùng KEYS).
- Keys: "status:data" TTL 2s (trạng thái monitor + CPU/RAM/Disk psutil);
  "events:list:{page}:{limit}:{ai_result|all}:{camera|all}:{status|all}"
  TTL 3600s; "recordings:list:{page}:{limit}:{camera|all}:{date_from|all}:
  {date_to|all}" TTL 3600s; "events:trends" TTL 3600s.
- Invalidation: mọi insert/update/xóa event -> xóa pattern "events:list:*",
  "recordings:list:*" và key "events:trends".

B4.2. SQLite — data/fall_detection.db, WAL mode, foreign_keys ON, timeout 10s
- Bảng events: id INTEGER PK AUTOINCREMENT; time TEXT NOT NULL (ISO UTC);
  time_local TEXT (ISO UTC+7 cố định); status TEXT NOT NULL; camera TEXT;
  confidence REAL; ai_result TEXT; ai_raw TEXT (description ngắn);
  ai_response TEXT (raw model); message TEXT; error TEXT; image_file TEXT;
  teldrive_image_id/teldrive_image_name/teldrive_image_path TEXT;
  teldrive_video_id/teldrive_video_name/teldrive_video_path TEXT.
  Index idx_events_time ON events(time DESC). Cột teldrive_* thêm bằng
  ALTER TABLE nếu thiếu (migrate).
- Status dùng trong code: started, stopped, verified, ai_error,
  telegram_sent, telegram_error, cooldown, teldrive_video_uploaded,
  teldrive_video_error, teldrive_image_error, record_error,
  stream_reconnect, monitor_error, restart_error, config_applied,
  recording_thumb.
- Bảng users: id PK; username TEXT UNIQUE NOT NULL; password_hash TEXT NOT
  NULL; created_at TEXT NOT NULL.
- Bảng settings: key TEXT PK; value TEXT NOT NULL; updated_at TEXT NOT NULL
  (lưu toàn bộ config serialize TEXT; cameras/prompts là JSON string).
- Giới hạn: MAX_EVENTS=5000, vượt thì xóa (vượt + 500) event cũ nhất kèm
  ảnh; ảnh event quá 24h bị xóa; ngẫu nhiên 5% mỗi lần lưu ảnh gọi
  delete_old_events(7 ngày) + dọn teldrive_cache. Ảnh event lưu nén OpenCV:
  resize nếu rộng >1280px, JPEG quality 70; tên
  "<UTC %Y%m%dT%H%M%S%f>_<status>.jpg" trong data/event_images.
- Migration một lần: events.jsonl -> SQLite (.jsonl.migrated); config.json
  -> bảng settings (.migrated).

B4.3. Auth
- Mật khẩu: bcrypt hashpw(gensalt()); bytes cắt tại 72 byte. verify
  checkpw, mọi exception -> False.
- Session: JWT (python-jose) HS256, payload {"sub": username, "exp"}.
  Secret ưu tiên: config/env "jwt_secret" -> file data/.secret_key -> tự
  sinh secrets.token_urlsafe(32) ghi vào file đó.
- Login: POST /auth/login (form username/password/remember). Đúng -> token
  hạn 8 giờ (720 giờ nếu remember), redirect "/", cookie "session" httponly
  samesite=lax (không secure — khuyên đặt sau reverse proxy HTTPS).
- require_auth: decode cookie; invalid: path "/api/" hoặc Accept JSON ->
  401; còn lại -> 302 /login.
- Khởi động: chưa có user -> tạo admin/admin. Đổi credential: POST
  /api/user/update -> update DB + cookie token mới 8h.
